# cn-index-journal

## 项目简介

`cn-index-journal` 是一个用于归档和发布多个独立 HTML 页面的仓库。  
这些页面以静态 HTML 文件的形式存放，不依赖于任何特定域名或后端服务。  
本仓库的主要目的是为某些信息提供一种结构化的存档与展示方式，方便查阅与维护。

## 目录说明

```
cn-index-journal/
├── index.html          # 仓库入口页面
├── pages/              # 存放各类独立 HTML 页面
├── assets/             # 页面引用的静态资源（CSS、JS、图片等）
└── README.md           # 本文件
```

- `index.html`：仓库根目录下的主页，可作为浏览所有归档页面的起点。
- `pages/`：核心目录，每个 HTML 文件代表一个独立的归档页面，文件命名建议简洁、无空格。
- `assets/`：用于存放页面共用或专用的样式、脚本与媒体文件，保持页面资源集中管理。

## 页面归档说明

- 所有页面均为纯前端 HTML 文件，可直接通过 `file://` 协议或静态托管服务（如 GitHub Pages）打开。
- 页面内容以信息归档与展示为目的，不包含交互式后台或动态数据源。
- 归档内容遵循“一个页面一个主题”的原则，便于独立引用与更新。
- 建议在页面内添加必要的说明文字，帮助读者理解归档内容的来源与背景。

## 维护说明

- 本仓库欢迎提交新的归档页面，但请确保内容不违反相关法律法规及 GitHub 服务条款。
- 新增页面时，请在 `pages/` 目录下创建文件，并在 `index.html` 中添加对应链接。
- 若需修改或删除已有页面，请保持仓库结构清晰，避免破坏其他页面引用。
- 资源文件请尽量放入 `assets/` 目录，避免在根目录堆积无关文件。
